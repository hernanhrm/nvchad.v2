## Files to Modify

Add `"rust"` to the `ensure_installed` list in the treesitter opts block.

### 2. `lua/configs/conform.lua` — Add rustfmt

### 3. `lua/plugins/mason.lua` — Add Rust tools

### 4. `lua/configs/lspconfig.lua` — Do NOT add rust_analyzer here
`rustaceanvim` manages `rust_analyzer` internally (like `flutter-tools` manages `dartls`). No changes needed.

### 5. `lua/plugins/rust.lua` — NEW FILE: rustaceanvim plugin
Create following the pattern of `flutter.lua` and `golang.lua`:

```lua
return {
  "mrcjkb/rustaceanvim",
  version = "^6",
  lazy = false,
  ft = "rust",
  init = function()
    vim.g.rustaceanvim = {
      server = {
        default_settings = {
          ["rust-analyzer"] = {
            checkOnSave = {
              command = "clippy",
            },
            inlayHints = {
              parameterHints = { enable = true },
              typeHints = { enable = true },
              chainingHints = { enable = true },
            },
            cargo = {
              features = "all",
            },
          },
        },
      },
      dap = {
        adapter = {
          type = "executable",
          command = vim.fn.stdpath "data" .. "/mason/bin/codelldb",
        },
      },
    },
  end,
}
```

Key decisions:
- `lazy = false` + `ft = "rust"` — loads only for Rust files but ensures config is available early
- `version = "^6"` — follows rustaceanvim's recommended versioning
- `checkOnSave = "clippy"` — uses clippy for linting on save
- `inlayHints` enabled — consistent with vtsls config pattern
- DAP adapter points to Mason-installed codelldb
- `vim.g.rustaceanvim` — uses the global config table approach (rustaceanvim's recommended way)

### 6. `lua/plugins/dap.lua` — No changes needed
`rustaceanvim` handles DAP configuration internally via its `dap` config option. codelldb integration is configured in `rust.lua` above.

### 7. `lua/mappings.lua` — Add Rust-specific keymaps
Add a Rust section with keymaps for rustaceanvim commands:

```lua
-- Rust keymaps (rustaceanvim)
map("n", "<leader>rr", "<cmd>RustLsp runnables<CR>", { desc = "Rust: Runnables" })
map("n", "<leader>rd", "<cmd>RustLsp debuggables<CR>", { desc = "Rust: Debuggables" })
map("n", "<leader>re", "<cmd>RustLsp expandMacro<CR>", { desc = "Rust: Expand Macro" })
map("n", "<leader>rc", "<cmd>RustLsp openCargo<CR>", { desc = "Rust: Open Cargo.toml" })
map("n", "<leader>rp", "<cmd>RustLsp parentModule<CR>", { desc = "Rust: Parent Module" })
map("n", "<leader>ri", "<cmd>RustLsp inlayHints toggle<CR>", { desc = "Rust: Toggle Inlay Hints" })
```

These use the `<leader>r` prefix to stay organized alongside existing `<leader>f` (Flutter) patterns.
