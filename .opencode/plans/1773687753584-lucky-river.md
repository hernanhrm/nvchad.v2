# Fix: Tree-sitter parsers not installed

## Root Cause

The Go Tree-sitter parser (and likely all non-bundled parsers) are **not installed**. The `~/.local/share/nvim/site/parser/` directory is empty.

Neovim 0.11.6 only ships with: `c`, `lua`, `markdown`, `markdown_inline`, `query`, `vim`, `vimdoc`. All other parsers (including `go`) must be installed by `nvim-treesitter`.

The `ensure_installed` list in `lua/plugins/init.lua` is correct, but the new `nvim-treesitter` rewrite no longer auto-installs parsers on startup. It only uses `ensure_installed` when `:TSInstallAll` is explicitly run.

Additionally, the opts `highlight = { enable = true, use_languagetree = true }` and `indent = { enable = true }` are **deprecated/ignored** by the current nvim-treesitter. They do no harm but are misleading dead code.

## Fix

### 1. Clean up deprecated opts in `lua/plugins/init.lua`

Remove `highlight` and `indent` from the treesitter opts since they are no longer used by the new nvim-treesitter. Keep only `ensure_installed`:

```lua
{
  "nvim-treesitter/nvim-treesitter",
  opts = {
    ensure_installed = {
      "vim", "go", "lua", "vimdoc", "html", "css",
      "javascript", "typescript", "tsx", "json", "jsonc",
      "python", "markdown", "markdown_inline", "bash",
      "yaml", "toml", "regex", "gdscript",
    },
  },
},
```

### 2. Install the parsers

After the config change, open Neovim and run:
```
:TSInstallAll
```

This will install all parsers listed in `ensure_installed`.

Alternatively, install just Go: `:TSInstall go`

## Files to modify

- `lua/plugins/init.lua` — remove deprecated `highlight` and `indent` opts from treesitter spec

## Verification

1. Open Neovim
2. Run `:TSInstallAll` to install all parsers
3. Open a Go file
4. Verify syntax highlighting shows full color differentiation (function calls, types, strings, package names all have distinct colors)
5. Run `:InspectTree` to confirm Tree-sitter is parsing the Go AST
6. Run `:Inspect` on a function name to see `@function.call` or similar treesitter capture groups
