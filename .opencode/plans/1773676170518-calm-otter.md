# Plan: Install miasma.nvim theme

## Summary
Install the [xero/miasma.nvim](https://github.com/xero/miasma.nvim) colorscheme as a lazy.nvim plugin and configure it as the active theme.

## Context
- The current theme is `kanagawa-dragon`, set via NvChad's base46 system in `lua/chadrc.lua`.
- miasma.nvim is a **standalone Vim/Neovim colorscheme** (not a NvChad base46 theme), so it must be loaded as a separate plugin and activated with `vim.cmd("colorscheme miasma")`.
- NvChad's base46 theme system will be left as-is (miasma will override it at runtime).

## Changes

### 1. Create `lua/plugins/miasma.lua`
Add a new plugin spec file following the repo's one-plugin-per-file convention:

```lua
return {
  "xero/miasma.nvim",
  lazy = false,
  priority = 1000,
  config = function()
    vim.cmd "colorscheme miasma"
  end,
}
```

- `lazy = false` ensures it loads at startup.
- `priority = 1000` ensures it loads before other plugins that might depend on highlight groups.
- `config` applies the colorscheme after the plugin loads.

**File:** `lua/plugins/miasma.lua`

### 2. No other changes needed
- `lua/chadrc.lua` does not need to be modified. The base46 theme value is irrelevant once miasma overrides it via `vim.cmd("colorscheme miasma")`.
- `init.lua` remains unchanged.

## Verification
1. Open Neovim: `nvim`
2. Run `:Lazy sync` to install the new plugin
3. Confirm the miasma colorscheme is active: `:colorscheme` should output `miasma`
4. Check for errors: `:checkhealth`
